# 咚雞點餐系統－店長操作手冊 v1.0.2

## 1. 進入管理後台

開啟網站後進入 `/admin`。桌面版使用左側導覽，手機版使用底部導覽。

## 2. 新增一間完全乾淨的分店

1. 進入「分店管理」。
2. 按「新增分店」。
3. 填寫分店名稱、完整地址、聯絡電話、預估時間、外送費與包裝費。
4. 按「建立並進入菜單設定」。

新分店會進入 `SETUP` 狀態，分區與菜品皆為空白，不會複製其他分店資料。

## 3. 建立菜單分區

1. 在「菜單管理」確認上方分店選擇器是正確分店。
2. 輸入分區名稱，例如「招牌主餐」、「小點」、「飲料」。
3. 按「新增分區」。

仍有菜品使用中的分區不能刪除，需先移動或刪除該分區內的菜品。

## 4. 新增菜品與圖片

1. 按「新增菜品」。
2. 填寫菜品名稱、分區、售價、簡介與完整說明。
3. 按「上傳菜品圖片」，支援 JPG、PNG、WebP，單檔上限 6 MB。
4. 系統會將圖片最長邊縮放至 900px 並轉成 WebP，降低瀏覽器儲存空間占用。
5. 可設定標籤、備用插圖、供應狀態、首頁主打與辣味標記。
6. 按「儲存菜品」。

新分店的菜品必須有圖片才能完成發布條件。

## 5. 發布分店

設定進度包含：

1. 建立分區。
2. 新增菜品。
3. 上傳圖片。
4. 發布營業。

全部完成後「發布分店」按鈕會啟用。發布後分店狀態由 `SETUP` 轉為 `ACTIVE`，顧客端分店選單與菜單會立即顯示該分店。

## 6. 暫停與恢復營業

在「分店管理」可將營業中的分店暫停。暫停後顧客端不再顯示該分店；按「恢復營業」後重新顯示。

狀態轉換：

`SETUP → ACTIVE → PAUSED → ACTIVE`

## 7. 分店資料隔離規則

每間分店都有獨立的：

- 菜單分區
- 菜品
- 圖片
- 售價
- 供應狀態
- 外送費與包裝費

修改其中一間分店不會覆蓋其他分店。顧客切換至不同分店時，尚未付款的購物車、一起點房間與付款中狀態會清除，已付款歷史訂單會保留。

## 8. 目前交付版本的資料範圍

本交付版使用瀏覽器 `localStorage` 保存分店、菜單、圖片、購物車與訂單，適合單機展示、店長操作驗收與同一瀏覽器來源內的完整流程。

正式多裝置營運時，必須將 `application/catalog-store` 與 `application/order-store` 的持久化層替換為伺服器資料庫與物件儲存，例如 Supabase PostgreSQL、Authentication、Realtime、Storage，並接入正式 LINE Pay。現有 Domain 狀態機與主要 UI 不需要重寫。
